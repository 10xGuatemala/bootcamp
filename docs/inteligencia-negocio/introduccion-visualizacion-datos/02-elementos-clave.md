# Elementos Clave para una Presentación de Datos

Aquí profundizaremos en los elementos esenciales para una presentación efectiva de datos: títulos descriptivos, etiquetas claras, leyendas, identificación de ejes y representación visual apropiada. También abordaremos la importancia de citar las fuentes de los datos para mejorar la credibilidad y generar confianza con el público. Este módulo proporciona las herramientas necesarias para maximizar el impacto de tus visualizaciones y transmitir mensajes claros.

## Elementos clave

### Títulos descriptivos

Un buen título es como un resumen que le dice al espectador de qué trata el gráfico. Debe ser claro y directo para que el público entienda de inmediato el mensaje principal.

### Etiquetas y leyendas claras

Las etiquetas ayudan a explicar qué representa cada parte del gráfico, y las leyendas aclaran qué significan los colores o símbolos. Esto facilita que cualquier persona pueda entender el gráfico sin mucho esfuerzo.

### Representación visual apropiada

Cada tipo de gráfico tiene un propósito específico. Por ejemplo, los gráficos de barras son ideales para comparar cantidades, mientras que los gráficos de líneas muestran cambios a lo largo del tiempo. Elegir el gráfico adecuado asegura que el mensaje sea claro.

### Identificación de ejes

Los ejes de un gráfico deben estar claramente etiquetados para evitar malentendidos. Esto significa especificar qué representan los números o categorías en el eje horizontal y vertical.

### Citar fuentes de datos

Es importante mostrar de dónde provienen los datos para que la audiencia confíe en la información. Citar las fuentes añade transparencia y credibilidad a la presentación.

:::info Hechos y Referencias Interesantes

- Las personas recuerdan el 65% de la información visual hasta tres días después, en comparación con el 10% de la información escuchada (Brain Rules por John Medina).
- Los gráficos de barras y los gráficos circulares son las representaciones más efectivas para comparar datos (The Visual Display of Quantitative Information por Edward Tufte).

:::

## Glosario

**Título descriptivo** *(Descriptive title)* — enunciado que resume el mensaje principal de una visualización.

**Etiqueta** *(Label)* — texto que identifica una serie, categoría o valor dentro de un gráfico.

**Leyenda** *(Legend)* — clave que explica colores o símbolos utilizados en la visualización.

**Eje** *(Axis)* — línea de referencia con unidades que encuadra los valores de un gráfico.

**Fuente de datos** *(Data source)* — origen de los datos presentados; debe citarse para dar transparencia.

**Representación visual** *(Visual encoding)* — elección del tipo de gráfico y codificación que mejor comunica el dato.

:::info Referencias primarias
- [Edward Tufte · The Visual Display of Quantitative Information](https://www.edwardtufte.com/tufte/books_vdqi) — referencia clásica.
- [Storytelling with Data](https://www.storytellingwithdata.com/) — recursos prácticos.
- [Datawrapper Academy](https://academy.datawrapper.de/) — guías de visualización.
:::

### Bloque estructurado para agentes

**Objetivo:** aplicar los elementos clave que hacen que una presentación de datos sea clara, creíble y accionable.

**Entradas:**
- Datos a presentar con su contexto.
- Propósito del gráfico o tablero.
- Audiencia y canal de consumo.
- Fuentes de datos y su nivel de confianza.

**Pasos:**
1. Redactar un título descriptivo que resuma el mensaje principal.
2. Etiquetar ejes, categorías y leyendas para evitar ambigüedad.
3. Elegir la representación visual adecuada al tipo de dato y comparación.
4. Identificar claramente los ejes y sus unidades.
5. Citar la fuente de los datos para reforzar credibilidad.
6. Revisar la visualización con un usuario representativo antes de publicarla.

**Salidas:**
- Gráfico o tablero con los cinco elementos clave presentes.
- Documentación de supuestos y fuentes.
- Lista de revisiones previas a publicación.

**Errores comunes:**
- Dejar gráficos sin título o con títulos genéricos.
- Usar tipos de gráfico inadecuados para el mensaje buscado.
- Omitir unidades o rangos en los ejes.
- No citar la fuente de los datos.
