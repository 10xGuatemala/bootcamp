# Sistemas de Diseño e Implementación

El módulo 3.1 explica **por qué** un diseño funciona; este explica **cómo se convierte en producto**. Aquí está el puente entre la intención visual y el código, y entre un equipo humano y un agente de IA que lo asiste.

Un sistema de diseño no es una guía de estilo bonita. Es la **infraestructura** que hace que cientos de pantallas se sientan como la misma aplicación, que un desarrollador nuevo entregue consistencia desde el primer día, y que un agente de IA genere UI que un revisor humano reconoce como "la nuestra".

## Qué cubre

- **Design tokens:** la fuente única de verdad para color, tipografía, espaciado y radios — versionada y exportable a cualquier plataforma.
- **Arquitectura de componentes:** cómo pensar en átomos, moléculas y organismos que luego se materializan en React, Vue o C#.
- **Handoff técnico:** cómo documentar un diseño para que desarrollo no tenga que adivinar márgenes, estados o comportamiento responsive.
- **UX writing:** el texto de la interfaz como parte del sistema — internacionalización, estados dinámicos y claridad para LLMs.
