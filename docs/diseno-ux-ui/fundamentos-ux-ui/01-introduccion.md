# Introducción a UX/UI

El diseño UX (User Experience) y UI (User Interface) nacen de la necesidad de mejorar cómo las personas interactúan con productos y tecnología. UX se originó en estudios de psicología y diseño industrial, donde se buscaba crear experiencias funcionales y agradables para los usuarios. UI, por su parte, surgió con el diseño visual y de interfaces gráficas, enfocado en hacer esas interacciones intuitivas y atractivas. Ambos conceptos se combinan para poner al usuario en el centro del diseño, creando productos eficientes, fáciles de usar y visualmente impactantes.

![intro](./img/intro-01.svg)

## Glosario

**UX** *(User Experience)* — disciplina que estudia y diseña la experiencia global de una persona al interactuar con un producto o servicio.

**UI** *(User Interface)* — capa visual e interactiva por la cual el usuario opera un producto digital.

**Diseño centrado en el usuario** *(User-centered design)* — enfoque que prioriza las necesidades, contexto y tareas reales del usuario en cada decisión de diseño.

**Usabilidad** *(Usability)* — medida de qué tan fácil, eficiente y satisfactorio resulta usar un producto para una tarea concreta.

**Interacción** *(Interaction)* — intercambio de acciones y respuestas entre usuario y producto a través de la interfaz.

:::info Referencias primarias
- [Nielsen Norman Group · UX articles](https://www.nngroup.com/articles/) — referencia de prácticas y heurísticas de UX.
- [Material Design](https://m3.material.io/) — sistema de diseño de Google.
- [Apple Human Interface Guidelines](https://developer.apple.com/design/human-interface-guidelines/) — referencia de diseño de interfaces.
:::

### Bloque estructurado para agentes

**Objetivo:** situar el origen de UX y UI para entender por qué ambos se combinan al diseñar productos centrados en el usuario.

**Entradas:**
- Producto o servicio que se está diseñando o rediseñando.
- Perfil del usuario objetivo.
- Expectativas del negocio respecto a experiencia y marca.
- Estado actual de las prácticas de diseño en el equipo.

**Pasos:**
1. Identificar qué aspectos corresponden a experiencia (UX) y cuáles a interfaz (UI).
2. Reconocer el origen de cada disciplina y cómo se complementan.
3. Evaluar si el equipo tiene cobertura en ambas disciplinas.
4. Acordar un lenguaje común entre diseño, desarrollo y negocio.
5. Definir el alcance inicial del trabajo de UX/UI en el producto.

**Salidas:**
- Marco de referencia común para el equipo.
- Brechas identificadas en cobertura UX o UI.
- Alcance inicial del trabajo de diseño.

**Errores comunes:**
- Tratar UX y UI como sinónimos.
- Asignar ambas disciplinas a una sola persona sin contexto.
- Omitir a los usuarios reales en decisiones de diseño.
- Separar UX y UI en silos que no se comunican.
